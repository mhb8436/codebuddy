## 테스트 흐름

### Phase 1: 관리자 - 문제 은행 설정

| # | 테스트 항목 | 예상 결과 | Pass/Fail |
|---|------------|----------|-----------|
| 1 | 관리자 로그인 | 로그인 성공 | ☐ |
| 2 | `/admin` 페이지 → "문제 은행" 탭 | 문제 은행 UI 표시 | ☐ |
| 3 | 언어: `javascript`, 트랙 선택 | 토픽 통계 테이블 표시 | ☐ |
| 4 | 토픽 행의 "문제 생성" 버튼 클릭 | 생성 모달 표시 | ☐ |
| 5 | Easy:2, Medium:2, Hard:1 설정 후 생성 | 작업 시작 메시지 | ☐ |
| 6 | 생성 작업 목록에서 상태 확인 | `completed` 상태 | ☐ |
| 7 | 토픽 클릭하여 문제 목록 조회 | 5개 문제 표시 | ☐ |
| 8 | 문제 선택 후 "승인" 클릭 | 상태 `approved`로 변경 | ☐ |
| 9 | 여러 문제 선택 후 "일괄 승인" | 모두 승인됨 | ☐ |

| # | 테스트 항목 | 예상 결과 | Pass/Fail |

| 15 | 정답 코드 작성 후 "제출" | "정답입니다!" 표시 | ☐ |
| 16 | 오답 코드 작성 후 "제출" | 피드백 표시 | ☐ |
| 17 | 문제 번호 클릭하여 이동 | 이전 답안 유지됨 | ☐ |
| 18 | "시험 종료" 클릭 | 최종 결과 표시 (점수, 등급) | ☐ |

[QuestionGenerator] Starting job ...
[QuestionGenerator] Job ... completed: X questions saved
